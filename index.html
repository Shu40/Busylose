    03
    Launch: Open the app and grant required permissions for window tracking.

    🔍 Mastering Core Features

    Window Tracking

    BusyLoss monitors active window focus changes to calculate precise usage time. It ignores background processes to ensure data accuracy.

    AI Insights

    Our local AI model analyzes your focus patterns and provides weekly 'Productivity Scores' with suggestions for improvement.

    🛡️ Security & Privacy First

    We believe your data belongs only to you. Here is how we protect it:

    • **Local Storage:** All app usage data is encrypted and saved only on your device.
    • **No Keylogging:** We never record keystrokes, screenshots, or any personal content.
    • **Offline Mode:** The core engine works 100% offline; no data is sent to our servers.

    ⚙️ Advanced Configuration

    Customize BusyLoss to fit your professional workflow.

    // Edit config.json to exclude apps
"exclude": ["SystemSettings.exe", "Taskmgr.exe"],
"idle_timeout": 300 // seconds before marking idle

    💻 Terminal & Manual Execution

    For advanced users and developers who prefer the command line.

    Running the App via CLI

    // Basic tracking mode
BusyLoss.exe --track

// Start minimized to tray
BusyLoss.exe --silent --minimized

// Export usage report to JSON
BusyLoss.exe --export --format=json --output=report.json

    Manual Website Hosting

    To run this project locally for development:

    # Using Python
python -m http.server 8000

# Using Node.js (serve)
npx serve .

    ❓ Help & Troubleshooting

    App not tracking focus?

    Ensure BusyLoss has "Accessibility" permissions and is not being blocked by aggressive antivirus software.

    High CPU usage?

    This can occur during AI model updates. It should stabilize within 30 seconds of launching.
